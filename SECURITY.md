# سیاست امنیتی TEOS

## گزارش آسیب‌پذیری

اگر یک آسیب‌پذیری امنیتی در TEOS کشف کرده‌اید، لطفاً آن را از طریق ایمیل به **security@teos.local** گزارش دهید. **از ایجاد Issue عمومی خودداری کنید.**

ما ظرف ۴۸ ساعت پاسخ می‌دهیم و حداکثر ظرف ۳۰ روز مشکل را برطرف می‌کنیم.

## نسخه‌های پشتیبانی شده

| نسخه | پشتیبانی |
|------|-----------|
| 1.0.x | ✅ فعال |
| < 1.0 | ❌ پایان یافته |

## بهترین روش‌های امنیتی برای کاربران

1. **رمزهای قوی**: از رمزهای طولانی و تصادفی برای دیتابیس، Redis و JWT استفاده کنید.
2. **به‌روزرسانی منظم**: همیشه آخرین نسخه TEOS را اجرا کنید.
3. **HTTPS اجباری**: هرگز از HTTP بدون TLS استفاده نکنید.
4. **محدودیت دسترسی**: فقط پورت‌های ضروری (۸۰، ۴۴۳) را باز بگذارید.
5. **مانیتورینگ**: از Prometheus و Grafana برای نظارت بر رفتار غیرعادی استفاده کنید.
6. **بکاپ**: بکاپ‌های منظم تهیه کنید و در مکان امن نگهداری کنید.
