# مستندات امنیتی TEOS

## اصول کلی
- اصل حداقل دسترسی (Principle of Least Privilege)
- رمزنگاری داده‌ها در حال حرکت (TLS) و در حال استراحت (Encryption at rest)
- Audit logging کلیه عملیات حساس

## احراز هویت
- **ربات تلگرام**: توکن ربات (محرمانه)
- **API**: JWT با زمان انقضا (Access Token 30 دقیقه، Refresh Token 7 روز)
- **2FA**: اختیاری برای ادمین‌ها

## مجوزها (RBAC)
| نقش | دسترسی‌ها |
|------|-----------|
| User | مشاهده موزیک، دانلود، خرید سرویس، کیف پول، تیکت |
| Music Admin | مدیریت آهنگ‌ها، آمار موزیک |
| Service Admin | مدیریت سرویس‌ها، سفارشات، تأیید پرداخت |
| Super Admin | مدیریت کاربران، مشاهده همه لاگ‌ها |
| Owner | مدیریت Tenant، پلاگین، بکاپ، به‌روزرسانی |

## Rate Limiting
- 100 درخواست در دقیقه به ازای هر کاربر/IP
- محدودیت ویژه برای عملیات حساس (تغییر رمز، شارژ کیف پول): 5 در ۱۰ دقیقه

## امنیت API
- CORS محدود به دامنه‌های مشخص شده
- محافظت CSRF (با توکن)
- اعتبارسنجی ورودی با Pydantic
- جلوگیری از SQL Injection با SQLAlchemy ORM

## امنیت داده‌ها
- رمزنگاری رمزهای عبور با bcrypt
- ذخیره نشدن لاگین‌های حساس
- پشتیبان‌ها رمزنگاری می‌شوند (اختیاری)

## مقابله با سوءاستفاده
- Anti-Spam (بر اساس کلمات کلیدی و تکرار)
- شناسایی و مسدودسازی IP مخرب
- محدودیت تعداد دفعات تلاش ناموفق (5 بار قفل موقت)

## Webhook Security
- امضای درخواست‌ها با HMAC-SHA256
- Whitelist IP های مجاز

## به‌روزرسانی امنیتی
- وابستگی‌ها با `pip-audit` بررسی می‌شوند
- به‌روزرسانی خودکار ماژور فقط در محیط staging

## گزارش حادثه امنیتی
در صورت کشف آسیب‌پذیری، لطفاً به security@teos.local ایمیل بزنید.
